## Summary
Fixed two critical threading bugs preventing file dialogs from working reliably on first invocation:

1. **NativeFileDialogService initialization timing**: Service was initialized as field initializer before InitializeComponent(), causing dispatcher capture race condition. Moved initialization to after InitializeComponent().

2. **WebView2 response posting**: Bridge handlers use ConfigureAwait(false), meaning responses post from worker threads. CoreWebView2.PostWebMessageAsJson() requires UI thread. Added dispatcher marshaling to Post() method to ensure responses always post from UI thread.

- `src\PanelNester.Desktop\Bridge\WebViewBridge.cs`: Added dispatcher check in Post(), changed ConfigureAwait(false) → ConfigureAwait(true)
