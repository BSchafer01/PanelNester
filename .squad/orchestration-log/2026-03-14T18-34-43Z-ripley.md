## Scope Definition & Decisions Made

**Decision: Phase 4 — Full Import Pipeline Design Review**

### Shippable Slice Narrowed To:
1. **XLSX import** — ClosedXML-based reader, same `ImportResponse` contract, first-sheet-only in v1
2. **Unified file import dispatcher** — Routes `.csv` → `CsvImportService`, `.xlsx` → `XlsxImportService` by extension
3. **Part row editing** — Update, delete, and add part rows after import with automatic re-validation
4. **Re-validation pipeline** — Runs the same validation rules on edited rows against current material library
5. **Bridge extensions** — New messages for unified import and row editing operations
6. **Import page UI hardening** — Inline editing controls, add/delete rows, filter by material and validation status, sort columns

### Seam Ownership Assignments:
- **Parker** — Domain contracts (`IPartEditorService`, `PartRowUpdate` DTO), shared validation extraction (`PartRowValidator`), service implementations (`XlsxImportService`, `FileImportDispatcher`, `PartEditorService`)
- **Bishop** — Bridge message constants and handlers (`import-file`, `update-part-row`, `delete-part-row`, `add-part-row`), file dialog filter updates, state management for edit operations
- **Dallas** — Import page UI: "Choose File" button, inline editing (text inputs + material dropdown), add/delete row buttons, client-side filter/sort controls
- **Hicks** — XLSX import tests, file dispatcher tests, part editor service tests, bridge round-trip tests, implementation sequence validation

### Non-Goals (Deferred to Later Phases):
- Undo/redo on edits
- Multi-file import or append-import
- Custom column mapping or header aliases
- Drag-and-drop file import
- `.xls` (legacy binary Excel) support
- CSV/XLSX export
