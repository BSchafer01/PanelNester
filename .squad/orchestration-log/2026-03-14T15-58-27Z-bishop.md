## Task Assigned
Fix desktop app Web UI content resolution to prefer built vertical-slice over bundled placeholder.

## Problem
The desktop app was loading `WebApp\index.html` placeholder when a real Web UI build existed at `src\PanelNester.WebUI\dist\index.html`. The resolver was checking the app bin directory before searching ancestor roots.

## Solution Implemented

### 1. Code Review & Analysis
- Examined `WebUiContentResolver.cs` resolution order
- Confirmed placeholder fallback was checked too early in the search
- Identified that resolver needed ancestor-first logic

### 2. Resolver Order Fix
Changed `WebUiContentResolver.ResolveContentPath()` to:
1. Search all ancestor directories up to repo root for `src\PanelNester.WebUI\dist\index.html`
2. If found, use real Web UI build
3. Only fall back to bundled `WebApp\index.html` if no real build exists

### 3. Test Coverage
Added focused tests in `WebUiContentResolverTests.cs`:
- Verify real dist folder is found before placeholder
- Verify fallback works when dist does not exist
- Confirm ancestor traversal stops at repo root

### 4. Validation

- ✅ `npm run build` — Web UI dist generated successfully
- ✅ Runtime proof — desktop app now outputs `src\PanelNester.WebUI\dist` as resolved path
